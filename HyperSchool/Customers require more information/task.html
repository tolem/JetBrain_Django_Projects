Description

You still have a lot to say to your clients. All educational products should be fully represented. Let's add two more pages. One will contain information about the course content, teaching methods, duration, and staff. The other page will contain something like CVs of your best mentors, administrators, moderators, etc. to guide your student through the program to the mountains of knowledge.

You already know how to create templates, so add two more to the templates' directory. Think of interesting headings and add any information that would be useful for your clients. Furthermore, we are going to display information from our models. To access the fields of the necessary model, you can use the so-called dot notation — {{ model name.field name }}. This page should contain the title of the course placed in <h2> tags, other information in <p> tags on the HTML-template, and list of teachers participating with links to the pages with information about them. So, the same thing for the page on teachers (teacher_details) but with the Teacher model (name, surname in <h2> tags) should be done.

The next step is to add two view classes for each page in views.py. Create the template_name and model fields in them.

Finally, pay attention to the path endings in urls.py. Use regular expressions to address the <pk> of each item in the database. As a result, URL should be generated as schedule/course_details/1 for the first course and consequentially (2, 3…) for the rest of courses. For the second page, URLs are schedule/teacher_details/1, schedule/teacher_details/2, and so on.

Don't forget to provide both new pages with links to go back to the main page with list of courses (main.html).

If you can easily switch from one page to another, let's continue with the next stage.

Objectives
  • Create new HTML templates for the course and teacher details pages,
  • Add links for new pages,
  • Add views for new pages,
  • Use regexes in the URL mapper to adjust it.
