Description

HyperJob is a recruitment agency founded in 1970. Company managers still prefer to communicate over the phone or email and search for employees in an old-fashioned way. It was an efficient strategy twenty years ago, but nowadays, potential employees prefer to apply for jobs online. The problem is that HyperJob still doesn't have a website, so they want you to develop one as soon as possible.

The website should allow recruiters to create vacancies and job seekers to create resumes.

Objectives

We need to safely store all the vacancies and resumes in a database. Your first task is to develop data models to manage the database tables.

Use the default settings of the project with the predefined SQLite database.
  • Throughout the project, we will need at least two models: Vacancy and Resume. Both of them should have description and author fields. The description field is a text field with no more than 1024 symbols, and the author field is a foreign key linked to the django.contrib.auth.models.User model.
  • Looking ahead, as our service is intended to be used both by employers and employees with different roles, it's a good practice to make two apps inside of our project (for example, resume and vacancy), so Vacancy and Resume models should be stored in different models.py files, each inside the proper app directory.
  • Don't forget to add new applications to the INSTALLED_APPS list in the settings.py file.

Hint! If you get stuck when creating the author field, check out an example provided in the docs.

  • Define Vacancy and Resume in the models.py modules in applications and migrate them to the database by executing the following commands in the terminal:
python3 manage.py makemigrations
python3 manage.py migrate
